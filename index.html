A JAVASCRIPT DSL FOR GENERATING CYPHER QUERIES

What is it?

WINGMAN is a Javascript DSL for generating cypher queries. It is designed to only support querying, not updating, deleting or any other operations that could modify data. WINGMAN supports the generation of standard (non-parameterized) as well as parameterized cypher in order to allow the database to better optimize queries. This is very much a work in progress. Contributions, forks, comments, etc. are welcome. Bug and features requests are tracked on the issues page.

Design

WINGMAN is designed as a fluent API. Its basic structures primarily mirror those defined by the cypher language and are described below.

In order to avoid collisions with other JS libraries, all classes, functions, etc reside in the Factgem.wingman namespace.

Match

Match describes the patterns and properties of nodes, relationships and paths that will be used to find information. Currently, a match can have just a start node or it can have a startNode, relationship and endNode. More complex match paths must be represented by multiple match objects. This may change in a future release.

Relationship

Relationship represents a relationship in the database.

Return

Return defines what data should be returned from the query and if only distinct results should be returned. Return clauses can also define ordering and support pagination of results.

Cypher

This is the top level object that will ultimately produce the desired cypher query. The toString method produces standard cypher text and the toParameterized method produces parameterized cypher. The parameters method prodcues an object that contains all of the parameters and their values, typically for use when you are using parameterized cypher.

Node

Node represents a node in the database.

Where

Where represents a clause that defines matching parameters for a match object. It supports the standard comparison operators in addition to checks for the existence or non-existence of a property.

Examples

Below are some example Cypher queries along with the WINGMAN code to generate them.

match (p:Person)-[r:hasResidentialAddress]->(pl:Place) where pl.city<>'westminster' return p;

var cypher = new FactGem.wingman.Cypher();
var match = new FactGem.wingman.Match()
.startNode(new FactGem.wingman.Node('p', 'Person'))
.relationship(
    new FactGem.wingman.Relationship('r','hasResidentialAddress','outgoing'))
.endNode(new FactGem.wingman.Node('pl', 'Place'));
match.where('pl', 'city').notEqual('westminster');
cypher.addMatch(match).andReturn().variable('p');

match (p:Person) where has(p.gender) return p;

var cypher = new FactGem.wingman.Cypher();
var match = new FactGem.wingman.Match();
match.startNode(new FactGem.wingman.Node('p', 'Person'))
.whereHasProperty('p','gender');
cypher.addMatch(match).andReturn().variable('p');

match (p:Person)-[r:hasResidentialAddress]->(pl:Place) where pl.city<>'westminster' return distinct p, pl.city order by p.familyName skip 10;

var cypher = new FactGem.wingman.Cypher();
var match = new FactGem.wingman.Match();
match.startNode(new FactGem.wingman.Node('p', 'Person'))
.relationship(
    new FactGem.wingman.Relationship('r', 'hasResidentialAddress','outgoing'))
.endNode(new FactGem.wingman.Node('pl', 'Place'))
.where('pl', 'city').notEqual('westminster');
cypher.addMatch(match).andReturn().variable('p').andReturn()
.distinctValues().variable('pl').property('city')
.skip(10).orderBy('p','familyName');

match (p:Person)-[r:hasResidentialAddress]->(pl:Place) where pl.city<>'westminster' return distinct p, pl.city order by p.familyName skip 10 limit 100;

var cypher = new FactGem.wingman.Cypher();
var match = new FactGem.wingman.Match();
match.startNode(new FactGem.wingman.Node('p', 'Person'))
.relationship(
    new FactGem.wingman.Relationship('r', 'hasResidentialAddress','outgoing'))
.endNode(new FactGem.wingman.Node('pl', 'Place'))
.where('pl', 'city').notEqual('westminster');
cypher.addMatch(match).andReturn().variable('p').andReturn()
.distinctValues().variable('pl').property('city')
.orderBy('p','familyName').skip(10).limit(100);

match (p:Person)-[r:hasResidentialAddress]->(pl:Place) where p.age>=30 return distinct p, pl.city order by p.familyName skip 10 limit 100;

var cypher = new FactGem.wingman.Cypher();
var match = new FactGem.wingman.Match();
match.startNode(new FactGem.wingman.Node('p', 'Person'))
.relationship(
    new FactGem.wingman.Relationship('r', 'hasResidentialAddress','outgoing'))
.endNode(new FactGem.wingman.Node('pl', 'Place'))
.where('p', 'age').greaterThanOrEqualTo(30);
cypher.addMatch(match).andReturn().variable('p').andReturn()
.distinctValues().variable('pl').property('city')
.orderBy('p','familyName').skip(10).limit(100);

Namespace

In order to avoid collisions with other JS libraries, all classes, functions, etc reside in the Factgem.wingman namespace.

Cypher

This is the top level object that will ultimately produce the desired cypher query. The toString() method produces the parameterized cypher text and the parameters method prodcues an object that contains all of the parameters and their values. It supports the following methods:

addMatch(match)
removeMatch(match)
addOptionalMatch(match)
removeOptionalMatch(match)
limitTo(limit)
andSkip(skip)
andReturn(return)
parameters()
toString()

Match

Match describes the patterns and properties of nodes, relationships and paths that will be used to find information. Currently, a match can have just a start node or it can have a starNode, relationship and and endNode. More complex match paths must be represented by multiple match objects. This may change in a future release. It supports the following methods:

startNode(startnode)
relationship(relationship)
endNode(endnode)
where(where)
toString()

Node

Node represents a node in the database. It supports the following methods:

addProperty(name,value)

Relationship

Relationship represents a relationship in the database. It supports the following methods:

addProperty(name,value)

Return

Return defines what data should be returned from the query. It supports the following methods:

variable(variable)
property(property)
andReturn()
orderBy(variable, property)
descending()
skip(skip)
limit(limit)
distinctValues()
countResults()
toString()

Where

Where represents a clause that defines matching parameters for a match object. It supports the following methods:

equals(value)
lessThan(value)
greaterThan(value)
notEqual(value)
lessThanOrEqual(value)
greaterThanOrEqualTo(value)
toString()
